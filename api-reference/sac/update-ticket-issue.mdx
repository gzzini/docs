---
title: "Atualizar Assunto do Ticket"
api: "PATCH https://sac-waffle-v2.theo-110.workers.dev/support/{id}/title"
description: "Atualiza o assunto (issue) de um ticket"
---

## Descrição

Atualiza o assunto/motivo (`issue`) de um ticket. O assunto deve ser um dos valores pré-definidos. Requer autenticação.

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token JWT para autenticação
</ParamField>

<ParamField header="Content-Type" type="string" required>
  `application/json`
</ParamField>

## Path Parameters

<ParamField path="id" type="number" required>
  ID do ticket
</ParamField>

## Body Parameters

<ParamField body="issue" type="string" required>
  Novo assunto do ticket

  Valores aceitos:
  - `Não recebi meus prêmios`
  - `Não estou recebendo a edição`
  - `Me desinscrevi e continuo recebendo`
  - `Alterações de cadastro`
  - `Não estou conseguindo indicar`
  - `Não estou conseguindo indicar o the news`
  - `Prêmio com defeito`
  - `Meus pontos no streak não estão contabilizando`
  - `Outros`
  - `Outro`
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indica se a operação foi bem-sucedida
</ResponseField>

<ResponseField name="message" type="string" required>
  Mensagem descritiva do resultado
</ResponseField>

<RequestExample>

```bash
curl -X PATCH "https://sac-waffle-v2.theo-110.workers.dev/support/5082/title" \
  -H "Authorization: Bearer seu_token_jwt" \
  -H "Content-Type: application/json" \
  -d '{
    "issue": "Não estou recebendo a edição"
  }'
```

</RequestExample>

<ResponseExample>

```json Success
{
  "success": true,
  "message": "Título do ticket atualizado com sucesso"
}
```

```json Error - Campo Obrigatório
{
  "success": false,
  "message": "O título do ticket é obrigatório"
}
```

```json Error - Valor Inválido
{
  "success": false,
  "message": "Título inválido. Títulos válidos são: Não recebi meus prêmios, Não estou recebendo a edição, ..."
}
```

```json Error - Não Encontrado
{
  "success": false,
  "message": "Ticket não encontrado"
}
```

</ResponseExample>
